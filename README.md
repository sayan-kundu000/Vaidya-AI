Vaidya AI is an AI-powered medical assistant web application built with a Flask backend. It leverages Google's Gemini AI to provide specialized conversational responses based on specialized medical personas and utilizes Google Text-to-Speech (gTTS) to read AI responses back to the user.

> **Disclaimer:** The information provided by Vaidya AI is generated by artificial intelligence and should not replace professional medical advice, diagnosis, or treatment. Always consult a certified healthcare professional before making any medical decisions or taking any medications.

## Features

- **Multi-Persona Medical AI:** Choose from a wide variety of specialist personas, including traditional medicine systems (Ayurveda, Siddha, Homeopathy) and comprehensive general practitioners (All-In-One Surgeon).
- **Gemini AI Integration:** Powered by `gemini-2.5-flash` model for high-quality, specialized responses.
- **Text-to-Speech (TTS):** Integrated audio playback system powered by `gTTS` to read the AI's responses out loud.
- **Mock Mode:** Operates safely in an offline/mock mode if no AI API key is configured.

## Tech Stack

- **Backend:** Python, Flask
- **AI Integration:** `google-genai`
- **Text-to-Speech:** `gTTS`
- **Environment Management:** `python-dotenv`

## Installation

1. **Clone or Navigate to the Directory:**
   ```bash
   cd medical-ai
   ```

2. **Create and Activate a Virtual Environment:**
   ```bash
   python -m venv .venv
   # On Windows
   .venv\Scripts\activate
   # On Linux/Mac
   source .venv/bin/activate
   ```

3. **Install Dependencies:**
   Ensure your virtual environment is activated, then install the required packages using the generated requirements file.
   ```bash
   pip install -r requirements.txt
   ```

## Configuration

1. Locate or create a `.env` file in the root `medical-ai` directory.
2. Add your Google Gemini API key to the environment file:
   ```env
   GEMINI_API_KEY=your_google_gemini_api_key_here
   ```
   *Note: If you leave the API key blank, the application will fallback to a functional Mock Mode for testing purposes.*

## Running the Application

1. Start the Flask server:
   ```bash
   python app.py
   ```
2. Open your web browser and navigate to:
   ```
   http://127.0.0.1:5000
   ```

## Project Structure

- `app.py`: Main Flask application handling routing, AI logic setup, and TTS integration.
- `requirements.txt`: List of required Python packages (`flask`, `google-genai`, `python-dotenv`, `gTTS`).
- `templates/`: Contains HTML files (e.g., `index.html`).
- `static/`: Contains static assets like CSS, JS, and temporarily stores generated audio files (in `static/audio`).
- `.env`: Environment variables file storing secure keys.
